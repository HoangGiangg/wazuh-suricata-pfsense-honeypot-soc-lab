\# Wazuh Agent Log Collection



\## Purpose



Wazuh Agents are installed on lab machines to collect security logs and forward them to the Wazuh Manager.



This allows Wazuh to centralize logs from Honeypot services, Suricata, Server-real, Mail/FTP services, and host-based monitoring components.



\## Agent Deployment



| Host | Agent Purpose |

|---|---|

| Honeypot | Collect Cowrie, Heralding, and Snare/Tanner logs |

| Suricata Collector | Collect Suricata logs forwarded from pfSense |

| Server-real | Collect authentication, web, auditd, and FIM logs |

| Mail/FTP Server | Collect mail, FTP, and authentication logs |



\## General Log Collection Flow



Log Source

\-> Wazuh Agent

\-> Wazuh Manager

\-> Wazuh Indexer

\-> Wazuh Dashboard



\## Honeypot Log Sources



| Service | Log Type | Purpose |

|---|---|---|

| Cowrie | SSH session logs | Detect SSH brute-force and commands |

| Heralding | Authentication logs | Detect FTP, SMTP, POP3, IMAP credential attacks |

| Snare/Tanner | Web request logs | Detect SQLi, XSS, Path Traversal, web probing |



\## Suricata Collector Log Sources



| Source | Purpose |

|---|---|

| Suricata EVE JSON | Network IDS alert analysis |

| Suricata alert logs | Detect port scan and suspicious traffic |



\## Server-real Log Sources



| Log Source | Purpose |

|---|---|

| auth.log | Detect SSH login failures and sudo activity |

| nginx access log | Detect web probing and suspicious requests |

| nginx error log | Detect web service errors and anomalies |

| auditd logs | Detect security-relevant system activity |

| FIM events | Detect unauthorized file changes |



\## Mail and FTP Log Sources



| Log Source | Purpose |

|---|---|

| mail.log | Monitor mail service activity |

| mail.err | Monitor mail service errors |

| vsftpd.log | Monitor FTP logins and file transfer activity |

| auth.log | Monitor authentication attempts |



\## Wazuh Agent Configuration Notes



The Wazuh Agent should be configured to monitor specific log files using local file collection.



For public GitHub documentation, configuration examples should be redacted and stored in:



configs/wazuh/



\## Validation Steps



1\. Deploy the Wazuh Agent.

2\. Confirm the agent appears as active in Wazuh Dashboard.

3\. Add log collection configuration.

4\. Restart the Wazuh Agent.

5\. Generate test events.

6\. Confirm events appear in Wazuh Dashboard.

7\. Verify alerts are generated by Wazuh rules.

\## Security Notes



\- Do not upload real client.keys.

\- Do not upload full production ossec.conf files.

\- Redact usernames, passwords, IP addresses, tokens, and emails.

\- Use documentation examples instead of raw sensitive configuration files.
